## システム概要

### システムの役割

UKPLUS Adminは、UKPLUS OsakaのiOSアプリ（生徒用）と連携し、管理者と教師がプライベートレッスンの予約管理、生徒管理、授業管理などを行うためのWebアプリケーションです。

### ユーザーロール

| ロール | 説明 | アクセス方法 |
|--------|------|------------|
| **管理者（admin）** | すべての機能にアクセス可能 | Webブラウザ |
| **教師（teacher）** | 自分の予約・空き枠管理のみ | Webブラウザ |
| **生徒（student）** | 予約・閲覧のみ | iOSアプリのみ |

**重要**: 生徒はこのWebアプリにはアクセスできません。生徒用の機能はすべてiOSアプリで提供されます。

### 📅 予約カレンダー（`/admin/calendar`）

プライベートレッスンの予約状況を週単位または月単位で確認・管理します。

#### 表示モードの切り替え

- **週表示**: 時間軸付きの詳細なスケジュール表示
- **月表示**: 月全体の授業予定を一覧表示

画面上部の「週表示」「月表示」ボタンで切り替えられます。

#### 基本操作

1. **週の切り替え（週表示モード）**
   - 左右の矢印ボタンで前週・翌週に移動
   - 「今週」ボタンで現在の週に戻る

2. **月の切り替え（月表示モード）**
   - 左右の矢印ボタンで前月・翌月に移動
   - 「今月」ボタンで現在の月に戻る

3. **教師フィルター**
   - 右上のドロップダウンで特定の教師のスケジュールのみ表示

3. **空き枠の追加**
   - 「+ 空き枠を追加」ボタンをクリック
   - モーダルが開く
   - 以下を入力：
     - **教師**: 担当教師を選択（必須）
     - **授業名**: 例「英会話レッスン」（任意）
     - **日付**: カレンダーから選択（必須）
     - **開始時間**: 時・分をそれぞれ1分単位で選択（9時〜21時）
     - **空き枠の長さ**: 時間・分をそれぞれ1分単位で選択
   - 「空き枠を作成」ボタンで確定

   **空き枠の考え方:**
   空き枠は「教師が授業可能な時間帯」を表します。生徒はiOSアプリから空き枠の範囲内で自由に授業時間を選択して予約します。予約後、残りの空き時間はそのまま空き枠として残ります。

4. **授業準備時間（バッファ）の自動確保**
   生徒が予約すると、授業後に自動的に「準備時間（バッファ）」が確保されます。
   - バッファ時間は設定ページ（`/admin/settings`）で変更可能（デフォルト: 10分）
   - 例: 空き枠 9:00〜17:00 に対して 10:00〜11:00 の授業が予約された場合
     - 9:00〜10:00 → 空き枠（そのまま残る）
     - 10:00〜11:00 → 予約済み（授業）
     - 11:00〜11:10 → 準備時間（自動確保、予約不可）
     - 11:10〜17:00 → 空き枠（そのまま残る）

5. **スロットの詳細確認**
   - カレンダー上のスロット（色付きのボックス）をクリック
   - 詳細モーダルが開く

#### スロットの色の意味

**週表示:**

| 色 | 意味 |
|---|------|
| 緑（Green） | 空き - 生徒が予約可能 |
| 青（Blue） | 予約済み - 生徒が予約済み |
| オレンジ（Orange） | キャンセル済み |
| 灰色（Gray） | クローズ - 予約不可 |

**月表示:**

各日付のセルに以下の情報が表示されます：
- 青い丸（●）: 予約済みの授業数
- 緑の丸（●）: 空き枠の数
- 灰色の丸（●）: クローズ済みの枠数
- 時間と生徒名（最大3件まで表示）
- 4件以上ある場合は「+N件」と表示

#### スロット詳細モーダルでできること

**予約が入っていない空きスロットの場合:**
- 「クローズする」: 生徒が予約できないようにする
- 「削除」: スロットを完全に削除

**予約済みスロットの場合:**
- Zoom URLの編集（「編集」ボタンをクリック）
- 「予約をキャンセル」: 予約をキャンセルし、スロットを空きに戻す

**クローズ済みスロットの場合:**
- 「開放する」: スロットを空き状態に戻す

### 👨‍🎓 生徒管理（`/admin/students`）

生徒の登録、受講状況の確認、受講可能な教師の設定を行います。

#### 生徒の追加

1. 「+ 生徒を追加」ボタンをクリック
2. 以下を入力：
   - **名前**: 例「山田太郎」（必須）
   - **メールアドレス**: 例「xxxx@gmail.com」（必須）
   - **パスワード**: 6文字以上（必須）
3. 「追加する」ボタンで確定

**自動的に作成される内容:**
生徒を追加すると、以下の内容で受講登録が自動的に作成されます：
- 登録回数: 8回
- 有効期限: 作成日から3ヶ月後
- ステータス: アクティブ
- 振替可能回数: 2回

これにより、生徒は追加後すぐに授業を予約できるようになります。回数や期限は後から編集可能です。

#### 生徒一覧の見方

テーブルには以下の情報が表示されます：

| 列名 | 説明 |
|------|------|
| 生徒名 | 登録されている名前 |
| メールアドレス | ログイン用メールアドレス |
| 受講可能な教師 | この生徒が予約できる教師（制限がある場合のみ表示） |
| 残り回数 | アクティブな受講登録の残り回数 |
| 有効期限 | アクティブな受講登録の有効期限 |
| ステータス | アクティブ / 登録なし |
| 操作 | 「詳細」ボタン |

#### 生徒詳細画面

「詳細」ボタンをクリックすると、詳細モーダルが開きます。

**基本情報セクション:**
- 名前
- メールアドレス
- 登録日

**受講可能な教師セクション:**
- 「編集」ボタンで教師の制限を設定
- 複数の教師を選択可能
- 何も選択しない場合は「すべての教師の授業を受講可能」
- 「制限を解除」ボタンで制限をクリア

**アクティブな受講セクション:**
- 現在有効な受講登録を表示
- 鉛筆アイコン（編集ボタン）をクリックして編集
- 編集できる項目：
  - **登録回数**: 回数券の総回数（クイック追加ボタン: +1, +2, +4, +8）
  - **有効期限**: カレンダーから選択（クイック追加ボタン: +1ヶ月, +3ヶ月, +6ヶ月）
  - **ステータス**: アクティブ / 期限切れ / 消化済み / 無効
- 「保存」ボタンで確定

**過去の受講セクション:**
- 期限切れ・消化済みの受講登録を表示

**受講状態の管理:**
- 「すべての受講を停止」ボタン:
  - アクティブな受講登録をすべて停止
  - 生徒は授業を予約できなくなります
  - アカウントは残るため、後から再開可能
- 「停止中の受講を再開」ボタン:
  - 停止中の受講登録を再びアクティブにする
  - 生徒は再び授業を予約できるようになります

**生徒の削除:**
- 「この生徒を削除」ボタンで削除可能
- 確認画面が表示されます
- **注意**: この操作は取り消せません（アカウントとすべてのデータが削除されます）

### 👨‍🏫 教師管理（`/admin/teachers`）

教師の登録、プロフィール編集を行います。

#### 教師の追加

1. 「+ 教師を追加」ボタンをクリック
2. 以下を入力：
   - **名前**: 例「山田太郎」（必須）
   - **メールアドレス**: 例「xxxx@gmail.com」（必須）
   - **パスワード**: 6文字以上（必須）
   - **権限**: 教師 / 管理者（必須）
3. 「追加する」ボタンで確定

#### 教師カードの見方

各教師がカード形式で表示されます：
- プロフィール画像
- 名前
- メールアドレス
- 権限バッジ（管理者の場合のみ表示）
- 専門分野（設定されている場合）
- 自己紹介（設定されている場合）

#### 教師情報の編集

1. 教師カードの「編集」ボタンをクリック
2. 編集モーダルが開く
3. 編集できる項目：
   - **プロフィール画像**: 
     - 「画像を選択」ボタンで画像をアップロード
     - JPEG/PNG形式、最大5MB
     - 「画像を削除」で削除可能
   - **表示名**: 生徒に表示される名前
   - **自己紹介**: プロフィール文（複数行可）
   - **専門分野**: 
     - テキストボックスに入力してEnterキーまたは「+」ボタンで追加
     - 例: ビジネス英語、IELTS、日常会話
     - 各タグの「×」ボタンで削除
4. 「保存する」ボタンで確定

#### 教師の削除

- 編集モーダル下部の「このアカウントを削除する」をクリック
- 確認画面が表示されます
- **注意**: この操作は取り消せません

### 🎫 受講管理（`/admin/enrollments`）

全生徒の受講状況、回数券の使用状況を一覧で管理します。

#### 統計カード

- 総受講登録数
- アクティブな受講数
- 総残り回数
- 期限切れ受講数

#### フィルター

- すべて
- アクティブ
- 期限切れ
- 消化済み

#### 受講一覧テーブル

| 列名 | 説明 |
|------|------|
| 生徒 | 生徒名とメールアドレス |
| タイプ | 月額登録 / 回数券 |
| 残り / 登録 | 残り回数 / 登録回数 |
| 使用済み | 使用済み回数 |
| 振替 | 残り振替回数 / 振替可能回数 |
| 有効期限 | 期限（7日以内の場合は赤字で表示） |
| ステータス | アクティブ / 期限切れ / 消化済み / 無効 |
| 操作 | 鉛筆アイコン（編集） |

#### 受講情報の編集

1. 鉛筆アイコンをクリック
2. 編集モーダルが開く
3. 編集できる項目：
   - **登録回数**: 回数券の総回数（クイック追加: +1, +2, +4, +8）
   - **有効期限**: カレンダーから選択（クイック追加: +1ヶ月, +3ヶ月, +6ヶ月）
   - **ステータス**: アクティブ / 期限切れ / 消化済み / 無効
4. 「保存する」ボタンで確定

**自動計算される項目:**
- 残り回数 = 登録回数 - 使用済み回数
- 振替可能回数 = 登録回数 ÷ 4（切り上げ）

### 📅 マイスケジュール（`/teacher/schedule`）

自分の空き枠と予約を週・月・一覧で管理し、**後からいつでも確認・編集**できます。

#### 表示モードの切り替え

- **週表示**: 時間軸付きの詳細なスケジュール表示
- **月表示**: 月全体の授業予定を一覧表示
- **一覧**: 今日から約8週間先までの枠を表形式で時系列表示（確認・編集に向いています）

画面上部の「週表示」「月表示」「一覧」ボタンで切り替えられます。

#### 空き枠を後から確認・編集するには

1. **一覧**を選ぶと、日付・時間・状態（空き／予約済み／閉鎖）・授業名・生徒が表で表示されます。各行の **「確認・編集」** で詳細を開けます。
2. **週表示**または**月表示**でも、枠（セル）をクリックすると同じ詳細が開きます。
3. 詳細画面（タイトル: **空き枠の確認・編集**）では、予約が入っていない枠について次が可能です。
   - **日時・授業名の編集**: 「編集する」から日付・開始時刻・枠の長さ・授業名を変更し「保存」
   - **閉鎖**: 予約を受け付けない状態にする（枠は残る）
   - **閉鎖を解除して空きに戻す**: 閉鎖中の枠を再び空きにする
   - **削除**: 枠そのものを削除
4. **予約が入っている枠**は、日時の変更・閉鎖・削除はできません（授業の「完了」「ノーショー」のみ変更可能）。予約の取り消しが必要な場合は管理者へ相談してください。

週表示では、枠の色が見づらい場合に **「週表示の色を自分で設定」** から背景色・文字色を変更して保存できます。

#### 基本操作

1. **週の切り替え（週表示モード）**
   - 左右の矢印ボタンで前週・翌週に移動
   - 「今週」ボタンで現在の週に戻る

2. **月の切り替え（月表示モード）**
   - 左右の矢印ボタンで前月・翌月に移動
   - 「今月」ボタンで現在の月に戻る

3. **一覧モード**
   - 表示期間の枠が表で並びます。「再読込」で最新データを取り直せます。

4. **空き枠の追加**
   - 「空き枠を追加」ボタンをクリック
   - モーダルが開く
   - 以下を入力：
     - **授業名**: 例「英会話レッスン」（任意）
     - **日付**: カレンダーから選択（必須）
     - **開始時間**: 時・分をそれぞれ1分単位で選択（追加時は9時〜21時の範囲）
     - **空き枠の長さ**: 時間・分をそれぞれ1分単位で選択
   - 「空き枠を作成」ボタンで確定

   **ポイント:** 空き枠は自分が授業可能な時間帯として設定してください。生徒がiOSアプリから予約すると、授業後にバッファ時間が自動確保され、残りの時間は空き枠として維持されます。追加後の日時変更は「確認・編集」から行えます（編集時は0時〜23時から選択可能）。

5. **スロットの詳細**
   - 週・月: カレンダー上の枠をクリック
   - 一覧: 「確認・編集」をクリック

#### スロットの色の意味

**週表示:**

| 色 | 意味 |
|---|------|
| 緑系（デフォルト） | 空き - 生徒が予約可能 |
| 青系（デフォルト） | 予約済み - 生徒が予約済み |
| 灰色系（デフォルト） | 閉鎖 - 予約不可 |

※ 週表示では「週表示の色を自分で設定」から色を変更できます。

**月表示:**

各日付のセルに以下の情報が表示されます：
- 青い丸（●）: 予約済みの授業数
- 緑の丸（●）: 空き枠の数
- 灰色の丸（●）: 閉鎖済みの枠数
- 時間と生徒名（最大3件まで表示）
- 4件以上ある場合は「+N件」と表示

#### スロット詳細モーダルでできること

**予約済みスロットの場合:**
- 日時情報の確認
- 生徒情報の確認（名前、メールアドレス）
- **授業ステータスの変更:**
  - 「完了」: 授業を完了状態にする（生徒の残り回数が1回減る）
  - 「ノーショー」: 生徒が欠席した場合（回数は消費される）

**空き・閉鎖のスロット（有効な予約がない場合）:**
- **編集する**: 日付・開始時刻・枠の長さ・授業名を変更して保存
- **この枠を閉鎖する**: 予約不可にする（枠は残る）
- **閉鎖を解除して空きに戻す**: 閉鎖のみのとき表示
- **空き枠を削除**: 枠を削除

**注意**: 有効な予約が入っているスロットは、日時変更・閉鎖・削除はできません。

## よくある質問（FAQ）

### Q1. 生徒のステータスを「アクティブ」に変更するには？

**A**: 生徒を追加すると、**自動的に「アクティブ」状態になります**。

生徒追加時に以下の内容で受講登録が自動作成されます：
- 登録回数: 8回
- 有効期限: 3ヶ月後
- ステータス: アクティブ
- 振替可能回数: 2回

既存の生徒のステータスを変更する場合：
1. 対象の生徒の「詳細」ボタンをクリック
2. 「アクティブな受講」セクションで鉛筆アイコン（編集ボタン）をクリック
3. 「ステータス」ドロップダウンから「アクティブ」を選択
4. 「保存」ボタンをクリック

---

### Q2. 生徒の受講回数を追加するには？

**A**: 2つの方法があります。

**方法1: 生徒管理ページから**
1. `/admin/students` にアクセス
2. 対象の生徒の「詳細」ボタンをクリック
3. 「アクティブな受講」セクションで鉛筆アイコンをクリック
4. 「登録回数」を変更（クイック追加ボタン: +1, +2, +4, +8も利用可能）
5. 「保存」ボタンをクリック

**方法2: 受講管理ページから**
1. `/admin/enrollments` にアクセス
2. 対象の受講登録の鉛筆アイコンをクリック
3. 「登録回数」を変更
4. 「保存する」ボタンをクリック

---

### Q3. 講師はどのように空き枠を管理しますか？

**A**: 講師は以下の方法で空き枠を管理します：

1. Webブラウザで `https://[デプロイ先URL]/teacher/schedule` にアクセス
2. 教師アカウントでログイン
3. 「空き枠を追加」ボタンから新しい空き枠を作成
4. カレンダー上でスロットをクリックして詳細確認・削除が可能

**注**: 講師専用のiOSアプリは存在しません。Webブラウザからアクセスしてください。

---

### Q4. 月表示のカレンダーはありますか？

**A**: はい、**月表示機能が実装されています**。

管理画面（`/admin/calendar`）と講師画面（`/teacher/schedule`）の両方で、画面上部の「週表示」「月表示」ボタンで切り替えられます。

**月表示でできること:**
- 月全体の授業予定を一覧表示
- 各日付に予約済み・空き・閉鎖の件数を表示
- 各日付の授業時間と生徒名を確認（最大3件まで）
- スロットをクリックして詳細を確認

**注意**: 生徒用iOSアプリには月表示機能はありません。

---

### Q5. 生徒が予約した授業を確認するには？

**A**: 以下の方法で確認できます：

**方法1: 予約カレンダーから**
1. `/admin/calendar` にアクセス
2. 青色（予約済み）のスロットをクリック
3. 詳細モーダルで生徒情報を確認

**方法2: 授業管理から**
1. `/admin/sessions` にアクセス
2. フィルターで「予約済み」を選択
3. 一覧から対象の授業を確認

---

### Q6. 授業が終わった後の処理は？

**A**: 授業終了後は必ずステータスを更新してください：

1. `/admin/calendar` または `/admin/sessions` から対象の授業を開く
2. 詳細モーダルで以下のいずれかを選択：
   - **「完了にする」**: 授業が正常に実施された場合
   - **「ノーショー」**: 生徒が欠席した場合
3. 確認画面で「OK」をクリック

**重要**: どちらの場合も生徒の残り回数が1回減ります。

---

### Q7. 予約をキャンセルするには？

**A**: 以下の手順でキャンセルできます：

1. `/admin/calendar` から対象のスロットをクリック
2. 詳細モーダルで「予約をキャンセル」ボタンをクリック
3. 確認画面で「OK」をクリック
4. スロットは自動的に「空き」状態に戻ります

**注**: キャンセルした予約は「キャンセル済み」ステータスとして記録されます。

---

### Q8. 特定の教師だけが担当できる生徒を設定するには？

**A**: 以下の手順で設定します：

1. `/admin/students` にアクセス
2. 対象の生徒の「詳細」ボタンをクリック
3. 「受講可能な教師」セクションで「編集」ボタンをクリック
4. 担当可能な教師にチェックを入れる
   - 「全選択」「全解除」ボタンも利用可能
5. 「保存」ボタンをクリック

**制限を解除する場合:**
- 「制限を解除」ボタンをクリックすると、すべての教師の授業を受講可能になります

---

### Q9. お知らせを特定の生徒だけに送るには？

**A**: 以下の手順で個別配信できます：

1. `/admin/announcements` にアクセス
2. 「+ お知らせを作成」ボタンをクリック
3. タイトルと本文を入力
4. 「配信対象」で「個別生徒」を選択
5. 検索ボックスで生徒を検索
6. 対象の生徒にチェックを入れる
7. 「作成する」ボタンをクリック

---

### Q10. 教師が自分の授業を確認するには？

**A**: 教師は以下の方法で確認できます：

1. 教師アカウントでログイン
2. `/teacher` ダッシュボードで今日の授業を確認
3. `/teacher/schedule` で週単位のスケジュールを確認
4. スロットをクリックして詳細を確認

---

### Q11. 生徒を削除せずに一時的に停止するには？

**A**: 生徒詳細モーダルの「受講状態の管理」セクションから停止できます：

1. `/admin/students` にアクセス
2. 対象の生徒の「詳細」ボタンをクリック
3. 「受講状態の管理」セクションまでスクロール
4. 「すべての受講を停止」ボタンをクリック
5. 確認画面で「OK」をクリック

**停止すると:**
- 生徒は授業を予約できなくなります
- アカウントは残るため、iOSアプリへのログインは可能
- 受講登録のステータスが「無効」になります
- 後から「停止中の受講を再開」ボタンで再開可能

**再開するには:**
- 同じ手順で「停止中の受講を再開」ボタンをクリック

**削除との違い:**
- **停止**: アカウントは残る、後から再開可能
- **削除**: アカウントとすべてのデータが削除される、取り消し不可

## トラブルシューティング

### ログインできない

**症状**: メールアドレスとパスワードを入力してもログインできない

**原因と対処法:**

1. **メールアドレス・パスワードが間違っている**
   - 入力内容を再確認してください
   - パスワードは大文字・小文字を区別します

2. **アカウントが作成されていない**
   - 管理者に連絡し、アカウントが作成されているか確認してください

3. **ロールが正しく設定されていない**
   - Firebase Consoleで `users` コレクションを確認
   - `role` フィールドが `admin` または `teacher` になっているか確認

---

### 空き枠を追加できない

**症状**: 「空き枠を作成」ボタンを押してもエラーが表示される

**原因と対処法:**

1. **日付・時間が過去になっている**
   - 未来の日時を選択してください

2. **Firebase接続エラー**
   - ブラウザのコンソールを確認
   - 開発者に連絡してください

3. **権限エラー**
   - Firestoreセキュリティルールを確認
   - 開発者に連絡してください

---

### 生徒の残り回数が減らない

**症状**: 授業を「完了」にしても生徒の残り回数が減らない

**原因と対処法:**

1. **受講登録（enrollment）が存在しない**
   - `/admin/enrollments` で対象の生徒の受講登録を確認
   - 存在しない場合は新規作成が必要

2. **ステータスが「アクティブ」ではない**
   - 受講登録のステータスを「アクティブ」に変更

3. **システムエラー**
   - ブラウザのコンソールを確認
   - 開発者に連絡してください

---

### カレンダーが表示されない

**症状**: カレンダー画面が真っ白、または読み込み中のまま

**原因と対処法:**

1. **データの読み込みエラー**
   - ページを再読み込み（F5またはCmd+R）
   - それでも解決しない場合は開発者に連絡

2. **Firebase接続エラー**
   - インターネット接続を確認
   - ブラウザのコンソールを確認

---

### 画像がアップロードできない

**症状**: 教師のプロフィール画像をアップロードできない

**原因と対処法:**

1. **ファイル形式が対応していない**
   - JPEG、PNG形式の画像を使用してください

2. **ファイルサイズが大きすぎる**
   - 5MB以下の画像を使用してください

3. **Firebase Storageの設定エラー**
   - 開発者に連絡してください
